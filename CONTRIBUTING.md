# Contributing

Contributions are welcome and will be fully credited.

## Pull Requests

- **[PSR-12 Coding Standard](https://www.php-fig.org/psr/psr-12/)** - The easiest way to apply the conventions is to run `composer format`.
- **Add tests** - Your patch won't be accepted if it doesn't have tests.
- **Document any change in behaviour** - Make sure the `README.md` is kept up-to-date.
- **One pull request per feature** - If you want to do more than one thing, send multiple pull requests.

## Running Tests

```bash
composer test
```

## Running Code Style Fixer

```bash
composer format
```
